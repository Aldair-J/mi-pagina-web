Bases Teóricas

Fundamentos conceptuales del marco teórico

🖥️
Sistemas de Información
Herramientas para recopilar, procesar y almacenar datos. Mejoran la eficiencia operativa y reducen errores administrativos en universidades.
🌐
Aplicaciones Web
Arquitectura cliente-servidor accesible mediante navegadores. Permite gestión centralizada sin instalación local adicional.
🗄️
Base de Datos Relacional
Núcleo técnico con tablas relacionadas (1NF, 2NF, 3NF). Garantiza integridad y trazabilidad de participantes, modalidades y periodos.
🔗
Centralización de Datos
Consolidación desde Excel disperso a repositorio único estructurado, mejorando acceso, control y consistencia de la información.
📊
Visualización de Datos
Representación gráfica que transforma datos complejos en información comprensible para análisis institucional y detección de tendencias.
⚡
Power BI (Business Intelligence)
Dashboards interactivos conectados a MySQL. Permite análisis por facultad, carrera, modalidad RPU/Bilateral y periodo académico.
🎯
Toma de Decisiones
Información organizada que apoya planificación estratégica. Permite identificar baja participación de carreras de Ingeniería en intercambios.
🎓
Movilidad Académica
Flujos entrantes/salientes de estudiantes, docentes y administrativos. Modalidades: RPU, Bilateral y otros convenios UAC.

Definición de Términos Clave

Glosario conceptual del sistema propuesto

Sistema de Información
Conjunto de elementos para recopilar, procesar y gestionar información institucional, optimizando decisiones y procesos. (Parra Payares, 2026)
Aplicación Web
Software accesible mediante navegador en arquitectura cliente-servidor, sin instalación local requerida. (Fernández & Guevara, 2020)
Base de Datos Relacional
Almacenamiento en tablas vinculadas por claves primarias/foráneas que garantiza integridad y trazabilidad. (Cotrino & Vega, 2017)
Centralización de Datos
Consolidación de información dispersa en un único sistema estructurado, eliminando duplicidades e inconsistencias. (Yildiz & Yildirim, 2022)
Dashboard
Panel visual interactivo que presenta información mediante gráficos e indicadores para monitoreo en tiempo real. (Silva Ramírez, 2021)
Power BI
Herramienta Microsoft de Business Intelligence para dashboards interactivos conectados a diversas fuentes de datos. (Silva Ramírez, 2021)
Movilidad Académica
Desplazamiento temporal de estudiantes, docentes o administrativos entre instituciones en modalidades RPU, Bilateral y otros convenios.
Normalización (1NF-3NF)
Proceso de organizar tablas para eliminar redundancias e inconsistencias, garantizando integridad de datos. (Cotrino & Vega, 2017)
Business Intelligence (BI)
Estrategias y herramientas que transforman datos operativos en información estratégica para decisiones organizacionales. (Yildiz & Yildirim, 2022)

Redacción de Antecedentes

Síntesis argumentativa y aporte a la investigación

1
Yildiz & Yildirim (2022)
Abordan la implementación de un sistema de información para gestión y análisis de movilidad estudiantil universitaria, destacando arquitecturas tecnológicas que integran herramientas de Business Intelligence para facilitar la interpretación de grandes volúmenes de información y apoyar la toma de decisiones institucionales.
→ Aporta fundamentos técnicos para el sistema web con BD centralizada e incorporación de Power BI para análisis en la UAC.
2
Fernández Gironza & Guevara Campo (2020)
Presentan el desarrollo de un sistema de información para gestión de movilidad e internacionalización en universidad colombiana, destacando la importancia de la integridad y consistencia de la información mediante bases de datos relacionales y mecanismos de control y validación.
→ Fundamenta el diseño de sistema web con BD estructurada, enfatizando la calidad de datos como elemento central.
3
Cotrino Rico & Vega Márquez (2017)
Presentan el desarrollo de un sistema de información web para movilidad académica usando arquitecturas cliente-servidor y herramientas de desarrollo web. Destacan metodologías ágiles y módulos funcionales para gestión de usuarios, registro de datos y seguimiento de procesos.
→ Valida directamente tecnologías web, BD relacionales y arquitecturas estructuradas como solución al problema de información dispersa en la UAC.
4
Parra Payares (2026)
Aborda el desarrollo de una solución informática para la Oficina de Relaciones Internacionales, enfatizando la arquitectura de software y estructuración de BD para facilitar el acceso y análisis de información. Describe técnicas para administración de datos garantizando calidad, pertinencia y usabilidad.
→ Aporta fundamentos para BD centralizada de movilidad y respalda la integración de Power BI como herramienta de visualización institucional.
5
Silva Ramírez (2021)
Presenta la implementación de análisis de datos con Power BI en institución educativa peruana, evidenciando procesos de integración, transformación y modelado de datos para dashboards interactivos. Destaca la consolidación de datos de distintas fuentes para generación de indicadores con filtros y segmentaciones.
→ Aporta fundamentos técnicos para Power BI como herramienta de visualización y análisis de movilidad por facultad, modalidad y periodo en la UAC.
6
Quiroz Barbadillo (2024)
Presenta el desarrollo de un sistema de información con BD como elemento central para organización, almacenamiento y consulta en procesos administrativos. Resalta la plataforma centralizada para reducir dispersión de registros e incorpora herramientas de visualización y análisis para reportes gráficos institucionales peruanos.
→ Valida en contexto peruano el sistema web con BD centralizada y justifica Power BI para patrones de participación por facultad y periodo.
